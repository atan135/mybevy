# 方圆灵构蓝图规则

## 文档定位

本文记录方圆灵构家园原型第一阶段使用的本地蓝图生成规则。

第一阶段目标是：在当前 Bevy 场景框架内，通过读取本地 RON 蓝图文件，在一个平面上生成由方块和球体组成的静态家园预览。

本文面向两类使用者：

- 人类开发者：据此实现蓝图读取、校验和渲染。
- 本地 Codex：据此生成或修改符合规则的 `home_preview.ron` 蓝图文件。

本文只覆盖第一阶段家园原型，不覆盖装备、技能、服务端存储、多人同步、真实 AI 接口和正式家园继承规则。

## 默认文件路径

第一阶段默认读取：

```text
project/assets/fangyuan/home_preview.ron
```

后续可扩展为多个蓝图文件：

```text
project/assets/fangyuan/blueprints/
  home_preview.ron
  fire_cave.ron
  wind_tower.ron
  water_garden.ron
```

第一阶段实现时可以只支持一个固定路径。

## 蓝图文件格式

蓝图使用 RON 格式。

推荐结构：

```ron
(
  version: "1",
  name: "fire_cave_preview",
  description: "火属性洞府预览",
  max_primitives: 1000,
  bounds: (
    width: 40.0,
    depth: 40.0,
    height: 20.0,
  ),
  primitives: [
    (
      kind: "cube",
      position: [0.0, 0.5, 0.0],
      size: [1.0, 1.0, 1.0],
      color: [0.85, 0.32, 0.12, 1.0],
    ),
    (
      kind: "sphere",
      position: [2.0, 1.0, 0.0],
      size: [1.0, 1.0, 1.0],
      color: [1.0, 0.58, 0.18, 1.0],
    ),
  ],
)
```

## 顶层字段

### `version`

必填字符串。第一阶段固定为：

```text
"1"
```

### `name`

可选字符串。用于显示或调试。

建议使用小写英文、数字、下划线，例如：

```text
"fire_cave_preview"
```

### `description`

可选字符串。用于说明蓝图意图。

### `max_primitives`

必填整数。第一阶段上限不能超过 `1000`。

如果文件内声明超过 `1000`，实现应按错误处理，不应静默生成。

### `bounds`

必填对象。定义蓝图允许使用的空间范围。

字段：

- `width`：X 轴总宽度。
- `depth`：Z 轴总深度。
- `height`：Y 轴最大高度。

第一阶段推荐默认：

```ron
bounds: (width: 40.0, depth: 40.0, height: 20.0)
```

### `primitives`

必填数组。每个元素表示一个方相或圆相几何体。

数量必须满足：

```text
0 <= primitives.len() <= min(max_primitives, 1000)
```

## 几何体字段

### `kind`

必填字符串，只允许：

- `"cube"`：方相，渲染为正方体或长方体。
- `"sphere"`：圆相，渲染为球体或椭球体。

第一阶段不允许圆柱、圆环、文字、贴图模型、自定义 mesh 或粒子。

### `position`

必填三元数组：

```ron
position: [x, y, z]
```

坐标规则：

- X/Z 平面表示家园地面位置。
- Y 表示高度。
- 地面是 `y = 0.0`。
- 几何体中心不能在地面以下。

推荐校验：

```text
-bounds.width / 2 <= x <= bounds.width / 2
0.0 <= y <= bounds.height
-bounds.depth / 2 <= z <= bounds.depth / 2
```

如果后续希望严格避免几何体穿出边界，可额外按 `size / 2` 收缩可用范围。

### `size`

必填三元数组：

```ron
size: [x, y, z]
```

含义：

- cube：长方体在 X/Y/Z 三轴上的尺寸。
- sphere：球体在 X/Y/Z 三轴上的缩放尺寸。三轴相同表示标准球体，不同表示椭球表现。

第一阶段推荐限制：

```text
0.1 <= size.x <= 5.0
0.1 <= size.y <= 5.0
0.1 <= size.z <= 5.0
```

如果想保持“球”的世界观纯度，Codex 生成 sphere 时应优先使用三轴相同的尺寸。

### `color`

必填四元数组：

```ron
color: [r, g, b, a]
```

范围：

```text
0.0 <= r <= 1.0
0.0 <= g <= 1.0
0.0 <= b <= 1.0
0.0 <= a <= 1.0
```

第一阶段建议 `a = 1.0`，避免透明物体带来额外排序和渲染复杂度。

## 坐标和摆放规则

第一阶段所有方圆灵构都堆砌在平面上方。

规则：

- 不生成地面以下的几何体。
- 不生成远离平面的漂浮主体，除非有明显支撑结构。
- 大型结构应由多个方块或球体组合，而不是单个巨大几何体。
- 主要结构应落在 `bounds` 范围内。
- 可允许少量装饰悬浮，但应符合方圆灵构世界观，例如悬浮圆核、风属性浮球、阵法节点。

## 颜色建议

颜色应服务世界观和可读性。

四属性参考：

- 地：灰、青灰、土黄、深绿。
- 火：红、橙、金、暗紫。
- 水：蓝、青、白、淡紫。
- 风：青绿、浅蓝、白、亮黄。

建议同一蓝图内控制主色数量，避免随机噪音。

第一阶段渲染优化会按颜色复用材质，因此 Codex 生成时也应尽量复用颜色。

## 数量和性能规则

第一阶段硬限制：

```text
primitives.len() <= 1000
```

建议生成规模：

- 小型预览：50 到 150 个。
- 中型洞府：150 到 400 个。
- 大型压力测试：400 到 1000 个。

Codex 默认不应生成满 1000 个，除非用户明确要求压力测试。

## 渲染优化要求

第一阶段实现应遵守：

- cube 共享一个 unit cube mesh。
- sphere 共享一个 unit sphere mesh。
- 材质按颜色缓存复用。
- 每个 primitive 只保留静态 Transform。
- 生成后不为单个 primitive 添加逐帧逻辑。
- 所有 primitive 挂到当前方圆灵构内容 root 下，方便一键清空。

第一阶段不要求实现 GPU instancing、mesh merge 或 LOD。

## 禁止事项

Codex 生成蓝图时不得：

- 生成非 `cube` / `sphere` 的 `kind`。
- 生成超过 `1000` 个 primitive。
- 生成负数尺寸。
- 生成地面以下主体结构。
- 生成超大单体几何体绕过数量限制。
- 生成透明度复杂堆叠作为主要结构。
- 生成文字、真实世界图标、外部模型路径或脚本。
- 生成需要游戏运行时执行代码的内容。

## 错误处理建议

第一阶段实现遇到非法蓝图时应：

- 不生成非法 primitive。
- 在日志中记录原因。
- HUD 可显示蓝图加载失败或被跳过的数量。
- 已存在预览不应因为新蓝图非法而崩溃。

具体错误提示和 UI 可后续实现。

## Codex 生成提示词模板

可使用以下提示词让本地 Codex 生成蓝图：

```text
请根据 docs/世界观/方圆灵构蓝图规则.md，生成 project/assets/fangyuan/home_preview.ron。

需求：
- 主题：火属性洞府
- 几何体数量：不超过 200
- 只允许 cube 和 sphere
- 范围：width=40, depth=40, height=20
- 颜色：以红、橙、金为主，少量暗色方块做底座
- 结构：中心有圆相炉心，周围有方相阵基和三层平台
- 不要生成地面以下几何体
- 不要生成透明复杂结构
```

生成后应检查：

- RON 语法有效。
- `primitives.len()` 未超过要求。
- 所有 `kind` 合法。
- 所有位置、尺寸和颜色在允许范围内。

## 第一阶段示例

最小可用示例：

```ron
(
  version: "1",
  name: "minimal_preview",
  description: "最小方圆灵构预览",
  max_primitives: 1000,
  bounds: (
    width: 40.0,
    depth: 40.0,
    height: 20.0,
  ),
  primitives: [
    (
      kind: "cube",
      position: [0.0, 0.5, 0.0],
      size: [4.0, 1.0, 4.0],
      color: [0.35, 0.36, 0.38, 1.0],
    ),
    (
      kind: "sphere",
      position: [0.0, 2.0, 0.0],
      size: [1.5, 1.5, 1.5],
      color: [1.0, 0.55, 0.12, 1.0],
    ),
  ],
)
```
