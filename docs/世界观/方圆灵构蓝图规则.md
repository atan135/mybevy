# 方圆灵构蓝图规则

## 文档定位

本文记录方圆灵构第一批本地 RON 蓝图生成规则、第四阶段已落地的开发期 Prefab / Palette 与 Scene Layout RON v1 生成规则、第五阶段已落地的审核和预算规则，以及第六至第八阶段已落地的静态渲染规模化与材质 profile 规则。

当前最小闭环目标是：在当前 Bevy 场景框架内，通过读取本地 RON 蓝图文件，生成一个玩家 Entity 内部的 `FangyuanPrimitiveSet`，并显示一个由一个方块身体和一个球体头部组成的最小方圆玩家。

当前家园和静态对象预览目标是：默认通过 Scene Layout 引用 Prefab / Palette，在一个平面上展开由方块和球体组成的静态家园预览。玩家、simple blueprint、Prefab 和 Scene Layout 展开结果共用同一套 primitive 字段，并统一编译到 `FangyuanPrimitiveSet`。

本文面向两类使用者：

- 人类开发者：据此实现蓝图读取、校验和渲染。
- 本地 Codex：据此生成或修改符合规则的 `minimal_player.ron`、`home_preview.ron`、`home_prefabs.ron`、`home_layout.ron` 等蓝图文件。

本文只覆盖第一批本地蓝图源格式、第四阶段开发期 layout/palette 源格式、第五阶段本地审核预算规则，以及第六至第八阶段 CPU merge / static instance 原型可消费的静态数据边界，不覆盖装备、技能、服务端存储、多人同步、真实 AI 接口和正式家园继承规则。

前三阶段明确不实现 Prefab、Scene Layout 拆分。第四阶段已落地开发期 Prefab / Palette 与 Scene Layout 拆分，第五阶段已落地审核 report、budget profile、finding、suggestion、家园 HUD 和日志接入；第六至第八阶段已落地静态 CPU merge、static instance shared-mesh prototype 和统一 `FangyuanMaterialProfile`。但仍明确不实现动态 VFX、技能规则层、正式 Chunk / LOD / AOI、发布期 Bake、蓝图缓存继承、正式 GPU instance buffer / custom render pipeline、联网同步、正式家园编辑器、蓝图持久化、装备挂点正式接入或任意 shader。

## 默认文件路径

当前最小闭环默认读取：

```text
project/assets/fangyuan/avatars/minimal_player.ron
```

最小玩家蓝图应作为对象层规则可用性的基准样例：

- 内容是一个最简易玩家外观。
- 玩家由一个 cube 身体和一个 sphere 头部组成。
- 文件内所有 `primitive.kind` 只能是 `"cube"` 或 `"sphere"`。


## 蓝图文件格式

蓝图使用 RON 格式。

当前最小玩家蓝图推荐结构：

```ron
(
  version: "1",
  name: "minimal_player",
  description: "最小方圆玩家外观",
  max_primitives: 2,
  bounds: (
    width: 2.0,
    depth: 2.0,
    height: 3.0,
  ),
  primitives: [
    (
      kind: "cube",
      position: [0.0, 0.75, 0.0],
      size: [0.9, 1.5, 0.6],
      color: [0.25, 0.45, 0.95, 1.0],
    ),
    (
      kind: "sphere",
      position: [0.0, 1.75, 0.0],
      size: [0.7, 0.7, 0.7],
      color: [0.95, 0.78, 0.55, 1.0],
    ),
  ],
)
```

静态对象或家园蓝图推荐结构：

```ron
(
  version: "1",
  name: "fire_cave_preview",
  description: "火属性洞府预览",
  max_primitives: 1000,
  bounds: (
    width: 40.0,
    depth: 40.0,
    height: 20.0,
  ),
  primitives: [
    (
      kind: "cube",
      position: [0.0, 0.5, 0.0],
      size: [1.0, 1.0, 1.0],
      color: [0.85, 0.32, 0.12, 1.0],
    ),
    (
      kind: "sphere",
      position: [2.0, 1.0, 0.0],
      size: [1.0, 1.0, 1.0],
      color: [1.0, 0.58, 0.18, 1.0],
    ),
  ],
)
```

## 顶层字段

### `version`

必填字符串。第一阶段固定为：

```text
"1"
```

### `name`

可选字符串。用于显示或调试。

建议使用小写英文、数字、下划线，例如：

```text
"fire_cave_preview"
```

### `description`

可选字符串。用于说明蓝图意图。

### `max_primitives`

必填整数。第一阶段上限不能超过 `1000`。

如果文件内声明超过 `1000`，实现应按错误处理，不应静默生成。

### `bounds`

必填对象。定义蓝图允许使用的空间范围。

字段：

- `width`：X 轴总宽度。
- `depth`：Z 轴总深度。
- `height`：Y 轴最大高度。

第一阶段推荐默认：

```ron
bounds: (width: 40.0, depth: 40.0, height: 20.0)
```

### `primitives`

必填数组。每个元素表示一个方相或圆相几何体。

数量必须满足：

```text
0 <= primitives.len() <= min(max_primitives, 1000)
```

RON v1 的最小必填字段仍是 `kind`、`position`、`size`、`color`。玩家、家园和静态对象都必须使用这组字段表达 primitive，不为家园或静态对象另开几何格式。当前 runtime 数据模型已经为每个 primitive 预留这些可选字段：

- `role`：语义角色，可选值为 `"structure"`、`"core"`、`"boundary"`、`"warning"`、`"trail"`、`"impact"`、`"decoration"`、`"socket"`、`"archive"`。
- `alpha`：透明度标量，范围 `0.0..=1.0`；省略时使用 `color` 的 alpha。
- `emissive`：发光强度，当前允许 `0.0..=16.0`；当前玩家预览不消费发光表现。
- `material_profile_id`：受控材质 profile 标识；省略表示默认材质 `material:default`。
- `lifecycle`：预留生命周期数据，可包含 `lifetime`、`spawn_tick`、`despawn_tick`；当前阶段不实现 VFX 播放或逐帧生命周期系统。

这些字段会编译进 runtime primitive 数据，不会让 primitive 变成玩法 Entity。当前 `FangyuanBlueprint` 是统一蓝图入口；玩家、家园和静态对象蓝图都应编译为 `FangyuanPrimitiveSet`。家园预览使用跳过非法 primitive 的编译路径，非法 primitive 不进入 runtime set，并通过 compile report 和 HUD 统计暴露。

## Prefab / Palette RON v1

Prefab / Palette 使用 RON v1 描述可复用方圆 primitive 组合。默认文件：

```text
project/assets/fangyuan/palettes/home_prefabs.ron
```

推荐结构：

```ron
(
  version: "1",
  name: "home_prefabs",
  description: "家园默认 prefab palette",
  max_primitives: 1000,
  bounds: (width: 40.0, depth: 40.0, height: 20.0),
  prefabs: [
    (
      id: "fence_segment",
      name: "fence_segment",
      pivot: [0.0, 0.0, 0.0],
      tags: ["home", "boundary"],
      primitives: [
        (
          kind: "cube",
          position: [0.0, 0.5, 0.0],
          size: [1.0, 1.0, 0.25],
          color: [0.55, 0.36, 0.18, 1.0],
        ),
      ],
    ),
  ],
)
```

Prefab / Palette 规则：

- `version` 固定为 `"1"`。
- `id` 使用小写 ASCII、数字和下划线，不能为空、重复、大小写混用、包含路径分隔符或危险字符。
- prefab 内 primitive 复用本文 `kind`、`position`、`size`、`color` 以及可选 `role`、`alpha`、`emissive`、`material_profile_id`、`lifecycle` 字段。
- prefab 可包含 `name`、`description`、`bounds`、`pivot`、`tags`、`max_primitives` 等元数据。
- prefab 不支持脚本、shader、服务端规则、外部资源引用、嵌套 prefab、战斗强度、资源产出或绕过预算的隐藏 primitive。
- prefab 和展开后的 primitive 都计入预算，不能绕过 `1000` expanded primitive 硬限制。

## Scene Layout RON v1

Scene Layout 使用 RON v1 描述 prefab instance。默认文件：

```text
project/assets/fangyuan/layouts/home_layout.ron
```

推荐结构：

```ron
(
  version: "1",
  name: "home_layout",
  description: "家园默认 scene layout",
  palette: "fangyuan/palettes/home_prefabs.ron",
  max_primitives: 1000,
  bounds: (width: 40.0, depth: 40.0, height: 20.0),
  instances: [
    (
      id: "front_fence_01",
      prefab: "fence_segment",
      position: [-8.0, 0.0, -10.0],
      scale: [1.0, 1.0, 1.0],
      tags: ["boundary"],
    ),
  ],
)
```

Scene Layout 规则：

- `version` 固定为 `"1"`。
- instance 必填 `prefab`、`position`、`scale`，可选 `id`、`name`、`tags`。
- `position` 必须是有限三元数组，并保持在 layout bounds 内。
- `scale` 必须是有限正数三元数组。
- `prefab` 必须引用 palette 中存在的 prefab id。
- `id` 如存在必须唯一。
- layout 只记录 prefab id、position、scale 和少量元数据，不支持 `rotation`、`material_override`、嵌套 layout 或 layout 引用 layout。
- 编译时 layout instance 展开为 `FangyuanPrimitiveSet`，并继续调用统一 primitive validator。
- 展开后非法 primitive 可跳过并记录 warning；顶层 layout/palette、缺失 prefab、非法 instance 和预算超限应返回结构化 error。

## 几何体字段

### `kind`

必填字符串，只允许：

- `"cube"`：方相，渲染为正方体或长方体。
- `"sphere"`：圆相，渲染为球体或椭球体。

当前规则不允许圆柱、圆环、文字、贴图模型、自定义 mesh 或粒子。

### 禁止旋转字段

方圆系统不支持旋转。蓝图不得出现这些字段或等价变体：

- `rotation`
- `quaternion`
- `euler`
- `angular_velocity`
- `rotate`
- `spin`

角色朝向、动作和技能方向后续应通过移动方向、局部 primitive 分布、scale 变化、生成和消失节奏表达，而不是通过旋转表达。

### `position`

必填三元数组：

```ron
position: [x, y, z]
```

坐标规则：

- X/Z 平面表示家园地面位置。
- Y 表示高度。
- 地面是 `y = 0.0`。
- 几何体中心不能在地面以下。

推荐校验：

```text
-bounds.width / 2 <= x <= bounds.width / 2
0.0 <= y <= bounds.height
-bounds.depth / 2 <= z <= bounds.depth / 2
```

如果后续希望严格避免几何体穿出边界，可额外按 `size / 2` 收缩可用范围。

### `size`

必填三元数组：

```ron
size: [x, y, z]
```

含义：

- cube：长方体在 X/Y/Z 三轴上的尺寸。
- sphere：球体在 X/Y/Z 三轴上的缩放尺寸。三轴相同表示标准球体，不同表示椭球表现。

第一阶段推荐限制：

```text
0.1 <= size.x <= 5.0
0.1 <= size.y <= 5.0
0.1 <= size.z <= 5.0
```

如果想保持“球”的世界观纯度，Codex 生成 sphere 时应优先使用三轴相同的尺寸。

### `color`

必填四元数组：

```ron
color: [r, g, b, a]
```

范围：

```text
0.0 <= r <= 1.0
0.0 <= g <= 1.0
0.0 <= b <= 1.0
0.0 <= a <= 1.0
```

第一阶段建议 `a = 1.0`，避免透明物体带来额外排序和渲染复杂度。

### `material_profile_id`

可选字符串，用于选择统一 `FangyuanMaterialProfile`。当前规则：

- 省略或填 `None` 时使用默认 profile `material:default`。
- 合法 id 长度必须为 `1..=64` 字节。
- 合法字符只允许 ASCII 字母、数字、`_`、`-`、`.`、`/`、`:`。
- 空字符串、超过长度上限或包含空格、中文、路径分隔外的特殊符号、脚本片段、URL 等内容均非法。
- 当前内置默认 profile 为 `material:default`。其他 profile id 可以作为受控表项或调试数据出现，但必须先通过审核和 profile registry 校验。
- 未知但语法合法的 profile 在材质解析时会 fallback 到默认 profile，并保留 requested id 和 fallback reason 供日志 / HUD / 测试定位；蓝图生成仍应优先使用已有受控 profile，不应依赖 fallback 表达正式效果。
- `material_profile_id` 不允许引用 shader、脚本、外部纹理、模型路径或任意材质程序。

材质合成规则：

- primitive 的 `color`、`alpha`、`emissive` 仍是 per-instance 输入。
- profile 提供 base color / alpha / emissive，以及 alpha、emissive 的合成和 clamp policy。
- CPU merge、static instance 和 standard render 都应保留 profile 语义；CPU merge 可以用顶点色承载多颜色并复用少量材质。
- 透明路径由合成后的 alpha 进入预算和渲染分组；不透明和透明内容不得混入同一静态合批路径。

## 坐标和摆放规则

第一阶段所有方圆灵构都堆砌在平面上方。

规则：

- 不生成地面以下的几何体。
- 不生成远离平面的漂浮主体，除非有明显支撑结构。
- 大型结构应由多个方块或球体组合，而不是单个巨大几何体。
- 主要结构应落在 `bounds` 范围内。
- 可允许少量装饰悬浮，但应符合方圆灵构世界观，例如悬浮圆核、风属性浮球、阵法节点。

## 颜色建议

颜色应服务世界观和可读性。

四属性参考：

- 地：灰、青灰、土黄、深绿。
- 火：红、橙、金、暗紫。
- 水：蓝、青、白、淡紫。
- 风：青绿、浅蓝、白、亮黄。

建议同一蓝图内控制主色数量，避免随机噪音。

第一阶段渲染优化会按颜色复用材质，因此 Codex 生成时也应尽量复用颜色。

## 数量和性能规则

当前 simple blueprint 和 layout 展开后的硬限制：

```text
primitives.len() <= 1000
expanded primitives <= 1000
```

建议生成规模：

- 小型预览：50 到 150 个。
- 中型洞府：150 到 400 个。
- 大型压力测试：400 到 1000 个。

Codex 默认不应生成满 1000 个，除非用户明确要求压力测试。

第五阶段默认审核预算还会给出推荐和硬限制：

- `hard_primitive_limit = 1000`，超过后为 error。
- `recommended_primitive_limit = 800`，超过后为 warning。
- bounds 默认最大 `64 x 64 x 64`。
- 单个 primitive 最大 extent 默认 `16`，单个 primitive 最大体积默认 `4096`。
- 总 primitive 体积默认最大 `32768`。
- alpha primitive 推荐不超过 `32`，硬限制 `128`。
- emissive primitive 推荐不超过 `24`，硬限制 `96`。
- emissive intensity 不能超过 runtime primitive 支持的最大值。
- material profile 推荐不超过 `8`，硬限制 `24`。

生成建议：

- 普通家园或洞府预览优先控制在 `800` 个 primitive 内。
- 透明和发光只用于核心、警示、灵核或少量装饰，不作为大面积主体结构。
- 复用颜色和 material profile，避免为每个 primitive 生成唯一材质参数。
- 大型结构优先拆成可复用 prefab 和 layout instance，但展开后的 primitive 仍计入 `1000` 硬限制。

## 渲染优化要求

当前实现应遵守：

- cube 共享一个 unit cube mesh。
- sphere 共享一个 unit sphere mesh。
- standard、CPU merge 和 static instance 三种家园预览渲染模式都必须消费统一 primitive 数据；默认仍使用 standard。
- CPU merge 只处理静态、可审核、无 lifecycle 的内容，并按 primitive kind、material profile、透明路径、颜色、发光和 source/debug 信息稳定分组。
- static instance 当前是 shared-mesh prototype，用于验证 instance array、batch key、buffer hash、HUD stats 和 fallback；它不是正式 GPU instance buffer / custom render pipeline。
- 材质通过 `FangyuanMaterialProfile`、per-instance color / alpha / emissive 和受控缓存复用表达，不按每个 primitive 线性创建唯一材质。
- primitive 是 `FangyuanPrimitiveSet` 内的 runtime 数据，字段包含 `kind`、`local_position`、`scale`、`color`、`role`、`alpha`、`emissive`、`material_profile_id` 和 `lifecycle`。
- 家园预览逻辑根持有 `FangyuanHomeObject`、`FangyuanPrimitiveSet` 和 `FangyuanObjectState`。
- 标准渲染适配层可以从 primitive 派生 render-only 子实体，并挂在对应逻辑 Entity 下方便清理；这些子实体只承载 mesh、material、visibility、transform 和显示标记。
- render-only 子实体不得挂载输入、移动、血量、技能、authority 或业务状态。
- 生成后不为单个 primitive 添加逐帧玩法逻辑。
- Reload、Clear 和 Exit 只应清理逻辑根和派生的 render-only 子实体，不应把 primitive 升级为独立玩法对象。
- HUD 统计应来自 `FangyuanPrimitiveSetStats`、compile report 和资产路径，不应从 render-only 子实体反推业务数量。
- 家园预览默认读取 `FANGYUAN_HOME_SCENE_LAYOUT_PATH` 和 `FANGYUAN_HOME_PREFAB_PALETTE_PATH`，展开后生成家园逻辑根。
- Reload 重新读取 layout/palette 并展开；Clear 只清空当前展开内容并保留基础空间；Exit 清理当前 session 下的展开内容和 stats。
- 默认 HUD 可显示 `layout loaded gen 138/1000 skip 0`、`pal 1 pf 5 used 5 inst 40 mat 15`。
- 第五阶段 HUD 还会显示 audit 状态、error / warning 数和主 code，例如 `audit passed e0 w0 -`、`audit warning e0 w1 invalid_primitive_color`、`audit failed e1 w0 missing_prefab`。
- 第六至第八阶段 HUD / 日志还会显示 render mode、material profile、opaque / transparent、emissive、material resource、static instance batch / count / buffer bytes 和 fallback reason 等调试统计。

第四阶段已实现开发期 Prefab / Palette 与 Scene Layout 拆分，第五阶段已实现审核和预算接入，第六至第八阶段已实现静态 CPU merge、static instance shared-mesh prototype 和统一材质 profile；当前仍不实现动态 VFX、技能规则层、正式 Chunk / LOD / AOI、发布期 Bake、蓝图缓存继承、正式 GPU instance buffer / custom render pipeline、联网同步、正式家园编辑器、蓝图持久化、装备挂点正式接入或任意 shader。

## 禁止事项

Codex 生成蓝图时不得：

- 生成非 `cube` / `sphere` 的 `kind`。
- 生成超过 `1000` 个 primitive。
- 生成负数尺寸。
- 生成地面以下主体结构。
- 生成 rotation、quaternion、euler、angular_velocity、rotate、spin 或其他旋转字段。
- 在 Scene Layout instance 中生成 rotation、方向字段、`material_override` 或嵌套 layout。
- 在 Prefab 中生成脚本、shader、服务端规则、外部资源引用或嵌套 prefab。
- 生成非法 `material_profile_id`，或通过 `material_profile_id` 引用 shader、脚本、外部资源、URL、贴图路径、模型路径或玩家任意材质程序。
- 生成超大单体几何体绕过数量限制。
- 生成透明度复杂堆叠作为主要结构。
- 生成文字、真实世界图标、外部模型路径或脚本。
- 生成需要游戏运行时执行代码的内容。
- 把动态 VFX、技能规则层、Chunk、LOD、AOI、Bake、缓存继承、联网同步、正式家园编辑器、装备挂点、正式 GPU instance buffer 或任意 shader 字段伪装成当前 checklist 可用能力。

## 审核规则和错误 code

第五阶段统一使用 `FangyuanAuditReport` 表达审核结果。Report 包含：

- `status`：`Passed`、`PassedWithWarnings`、`Failed`。
- `summary`：error、warning、info、palette、prefab、instance、generated、skipped、cube、sphere、color、material、alpha、emissive、lifecycle、role distribution 和 top-level/layout/palette valid 等统计。
- `findings`：每条 finding 包含 `severity`、`code`、`field_path`、`reason`、`source_kind`，并可定位 primitive、prefab、instance 和 prefab primitive。
- `suggestions`：每条 suggestion 包含降级动作、字段路径、原因和预估效果。

常见 source kind：

- `Blueprint`：simple blueprint。
- `PrefabPalette`：Prefab / Palette。
- `SceneLayout`：Scene Layout。
- `RuntimePrimitiveSet`：展开后的 runtime primitive set 预算面。

常见 blueprint code：

- `unsupported_version`
- `primitive_count_exceeded`
- `invalid_bounds_dimension`
- `invalid_primitive_position`
- `primitive_below_ground`
- `invalid_primitive_size`
- `invalid_primitive_color`
- `invalid_primitive_alpha`
- `invalid_primitive_emissive`
- `invalid_primitive_material_profile`
- `invalid_primitive_lifecycle`

`invalid_primitive_material_profile` 常见原因：

- `material_profile_id` 为空字符串。
- `material_profile_id` 超过 `64` 字节。
- `material_profile_id` 含有非 ASCII 字母、数字、`_`、`-`、`.`、`/`、`:` 的字符。
- 试图把 shader、脚本、外部路径或 URL 塞进 profile id。

常见 prefab / palette code：

- `unsupported_version`
- `invalid_palette_bounds`
- `palette_primitive_budget_exceeded`
- `invalid_prefab_id`
- `invalid_prefab_bounds`
- `invalid_prefab_pivot`
- `invalid_prefab_tag`
- `prefab_primitive_budget_exceeded`
- `total_primitive_budget_exceeded`
- `invalid_prefab_primitive`

常见 scene layout code：

- `unsupported_version`
- `invalid_layout_bounds`
- `layout_primitive_budget_exceeded`
- `invalid_palette_path`
- `invalid_instance_prefab_id`
- `missing_prefab`
- `invalid_instance_id`
- `invalid_instance_tag`
- `invalid_instance_position`
- `invalid_instance_scale`
- `expanded_primitive_budget_exceeded`

常见 runtime budget code：

- `primitive_count_above_recommended`
- `primitive_count_above_hard_limit`
- `bounds_above_limit`
- `primitive_extent_above_limit`
- `primitive_volume_above_limit`
- `total_volume_above_limit`
- `alpha_count_above_recommended`
- `alpha_count_above_hard_limit`
- `emissive_count_above_recommended`
- `emissive_count_above_hard_limit`
- `emissive_intensity_above_limit`
- `material_profile_count_above_recommended`
- `material_profile_count_above_hard_limit`

降级建议动作：

- `ReducePrimitives`：减少 primitive、generated 或 expanded 数量。
- `ShrinkBounds`：收缩 bounds、体积或单体尺寸。
- `RemoveAlpha`：移除或减少透明 primitive。
- `LowerEmissive`：降低发光数量或强度。
- `ReplaceMaterialProfile`：减少 material profile 分支。
- `ReduceWarningRole`：减少 warning role 内容压力。

## 错误处理建议

当前实现遇到非法蓝图、非法 layout/palette 或非法 primitive 时应：

- 不生成非法 primitive。
- 在日志中记录原因。
- compile report 记录 authored、palette、prefab、used prefab、instance、generated、skipped、material、top-level/layout/palette valid 和跳过原因。
- audit report 记录 `status`、`summary`、`finding` 和 `suggestion`；error / warning 尽量提供 `code`、`field_path`、`reason`。
- 单个展开后非法 primitive 可跳过并 warning；缺失 prefab、非法 instance、预算超限、顶层 layout/palette 非法应作为结构化 error。
- HUD 可显示蓝图加载失败、跳过数量、audit 状态、error / warning 数和主 code。
- 日志应输出 audit summary、finding、suggestion，便于定位生成提示词或资源文件问题。
- 已存在预览不应因为新蓝图非法而崩溃。

更完整的错误解释 UI、玩家可读文案和正式试炼场仍是后续能力。

## Codex 生成提示词模板

可使用以下提示词让本地 Codex 生成蓝图：

```text
请根据 docs/世界观/方圆灵构蓝图规则.md，生成 project/assets/fangyuan/home_preview.ron。

需求：
- 主题：火属性洞府
- 几何体数量：不超过 200
- 只允许 cube 和 sphere
- 范围：width=40, depth=40, height=20
- 颜色：以红、橙、金为主，少量暗色方块做底座
- 结构：中心有圆相炉心，周围有方相阵基和三层平台
- 不要生成地面以下几何体
- 不要生成透明复杂结构
- 不要生成 rotation、quaternion、euler、angular_velocity、rotate 或 spin
- material_profile_id 省略时使用 material:default；如需填写，只使用受控 ASCII id，不引用 shader、脚本、外部贴图或模型
- 不要生成动态 VFX、技能规则层、Chunk、LOD、AOI、Bake、缓存继承、联网同步、装备挂点、正式 GPU instancing 或任意 shader 字段
- 尽量控制在推荐预算内：primitive 少于 800，alpha 少于 32，emissive 少于 24，material profile 少于 8
```

生成后应检查：

- RON 语法有效。
- `primitives.len()` 未超过要求。
- 所有 `kind` 合法。
- 所有位置、尺寸和颜色在允许范围内。
- 透明、发光和 material profile 没有超过推荐预算。
- `material_profile_id` 缺省或为合法 ASCII id；缺省等价于 `material:default`，空字符串非法。
- 若 HUD 或日志出现 audit warning / failed，按主 code 和 finding 的 `field_path` 调整蓝图。
