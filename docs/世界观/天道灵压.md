# 天道灵压

## 文档定位

本文记录区域玩家聚集过多时，世界通过天道灵压、群修天劫或界域震荡进行分流的世界观设计口径。

本文是设计文档，不代表区域密度检测、服务端负载调度、天劫表现、强制驱散或收益衰减已经实现。落地时应同步拆分为世界规则、服务端负载、场景表现、UI 提示和玩家保护规则。

## 核心概念

修行世界中，过多修士长期聚集在同一区域，会扰动地脉、灵气和天道稳定性。

当某个区域内玩家密度、战斗频率、技能释放、交易行为、宗门集结或实体数量超过阈值时，天道会逐步施加灵压。严重时，灵压会演化为群修天劫或界域震荡。

这个机制有两个目的：

- 世界观目的：修士不能无限聚集在同一个地方，需要分散探索和开拓世界。
- 技术目的：降低单一区域服务端压力，避免热点区域承载过高。

## 推荐命名

主机制推荐命名为：

- 天道灵压。

严重阶段可使用：

- 群修天劫。
- 界域震荡。
- 地脉过载。
- 灵气紊乱。

天道灵压适合作为常态压力；群修天劫适合作为高强度表现。

## 触发因素

天道灵压不应只由人数触发，可以综合判断区域压力。

可参考因素：

- 同区域玩家数量。
- 单位时间技能释放次数。
- 战斗结算频率。
- 掉落物、召唤物、灵构实体数量。
- 宗门或联盟集结规模。
- 交易和交互频率。
- 区域服务端负载。
- 当前世界事件状态。

同样的人数在主城、安全区、战场、秘境中的阈值可以不同。

## 分级表现

天道灵压应采用渐进式处理，而不是突然惩罚。

### 轻度灵压

用于提示玩家区域过载。

可能效果：

- 天道警示。
- 环境异象。
- 局部灵气波动。
- UI 提示当前区域拥挤。

### 中度灵压

用于降低继续聚集的收益。

可能效果：

- 区域收益下降。
- 部分技能消耗增加。
- 资源刷新效率降低。
- 传送进入冷却增加。
- 提示附近可迁移区域。

### 重度灵压

用于引导玩家主动离开。

可能效果：

- 群修天劫预兆。
- 部分非关键交互暂时受限。
- 战斗压力上升。
- 区域内持续警示。
- 开启临时分流入口。

### 群修天劫

用于极端拥堵或高压事件。

可能效果：

- 天劫落雷或界域震荡表现。
- 强制打断部分聚集行为。
- 将部分玩家引导到镜像区域、相邻区域或安全区。
- 临时关闭非必要高频交互。

群修天劫不应用于恶意秒杀普通玩家。它应优先承担分流和限压职责。

## 安全区规则

安全区可以触发天道灵压，但惩罚应更柔和。

安全区内更适合使用：

- 提示。
- 镜像分流。
- 交互排队。
- 非关键特效降级。
- 部分展示实体隐藏。

安全区不应轻易触发高伤害天劫。

## 与世界开拓的关系

天道灵压应引导玩家探索其他区域。

当某一区域压力过高时，系统可以同时：

- 提示附近低压力区域。
- 开启临时传送门。
- 增加边缘区域探索奖励。
- 生成天道自演的引导 NPC。
- 触发世界事件，把玩家导向多个地点。

玩家感知上应是“天道推动开拓”，而不是“系统粗暴赶人”。

## 平衡原则

天道灵压需要遵守这些原则：

- 提前预警，不突然惩罚。
- 优先降收益和限交互，再考虑强表现。
- 安全区惩罚更柔和。
- 不能把必要主线、回流和新手流程锁死。
- 不能永久损伤角色资产。
- 应提供迁移方向或替代区域。
- 技术限压要包装成世界规则，但不能牺牲基本体验。

## 待补充设计

后续需要继续补充：

- 区域压力计算规则。
- 不同场景的灵压阈值。
- 安全区和非安全区差异。
- 群修天劫表现和分流规则。
- 和服务端负载监控的映射关系。
- 玩家提示、传送和补偿规则。
